# Costa Beach Design System

Welcome to the Costa Beach design system documentation. This design system follows atomic design principles to create a consistent and maintainable UI across our application.

## Design Principles

1. **Consistency**: Components maintain consistent styling and behavior across the application
2. **Accessibility**: All components are built with accessibility in mind, following WCAG 2.1 guidelines
3. **Responsiveness**: Mobile-first approach with responsive design patterns
4. **Dark Mode**: Full support for light and dark themes

## Component Structure

Our components are organized following atomic design principles:

### Atoms
Basic building blocks of the interface:
- Button
- Input
- Heading
- Paragraph
- Icon

### Molecules
Combinations of atoms forming functional pieces:
- Form (Input + Button)
- Card (Heading + Paragraph)
- NavItem (Icon + Text)
- TextField (Input variation)

### Organisms
Larger composite components:
- Header
- Footer
- PropertyShowcase
- OwnerPortalSidebar
- DocumentList

### Templates
Page layouts:
- PublicLandingTemplate
- OwnerDashboardTemplate

### Pages
Complete pages:
- HomePage
- PropertyDetailPage
- ContactPage
- OwnerLoginPage
- OwnerDashboardPage

## Accessibility Guidelines

1. **Keyboard Navigation**
   - All interactive elements must be focusable
   - Logical tab order
   - Visible focus indicators

2. **Screen Readers**
   - Proper ARIA labels
   - Semantic HTML
   - Alternative text for images

3. **Color Contrast**
   - WCAG 2.1 AA compliant contrast ratios
   - Color is not the only means of conveying information

4. **Responsive Design**
   - Mobile-first approach
   - Proper viewport meta tags
   - Touch targets at least 44x44px

## Theme System

Our theme system uses CSS variables and Tailwind CSS for consistent styling:

```css
:root {
  --background: 0 0% 100%;
  --foreground: 222.2 47.4% 11.2%;
  /* ... other variables ... */
}

.dark {
  --background: 224 71% 4%;
  --foreground: 213 31% 91%;
  /* ... other dark mode variables ... */
}
```

## Getting Started

1. Install dependencies:
```bash
npm install
```

2. Run Storybook:
```bash
npm run storybook
```

3. View components in your browser at `http://localhost:6006`

## Contributing

When adding new components:

1. Create component in appropriate directory (`atoms`, `molecules`, etc.)
2. Add TypeScript interfaces for props
3. Include comprehensive Storybook stories
4. Add accessibility features
5. Test in both light and dark modes
6. Document usage examples
